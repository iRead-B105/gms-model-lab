# GMS Model Lab

GMS의 이미지·텍스트·TTS 모델을 같은 화면에서 실행하고 응답 시간과 크레딧을 비교하는 로컬 테스트 도구입니다.

## 시작하기

Node.js 22.13 이상과 pnpm이 필요합니다.

```powershell
pnpm install
pnpm dev
```

브라우저에서 `http://localhost:3000`을 연 뒤 아래 순서로 사용하면 됩니다.

1. 화면에 GMS 키를 입력하고 **키·전체 모델 확인**을 누릅니다.
2. 이미지, 텍스트 또는 TTS 모드를 선택합니다.
3. 모델, 프롬프트, 파라미터를 설정합니다.
4. 반복 횟수와 동시 요청 수를 정한 뒤 테스트를 시작합니다.
5. 필요하면 PNG·JPEG·WebP 이미지를 컨텍스트로 추가합니다.
6. 결과 화면과 하단의 로컬 기록에서 지연시간, 토큰, 실제 차감 크레딧을 비교합니다.

GMS 키는 기본적으로 현재 페이지에서만 사용합니다. **이 브라우저에 키 저장**을 누르면 같은 주소로 다시 접속했을 때 자동으로 불러오며, **저장 키 삭제**로 언제든 제거할 수 있습니다. 키는 브라우저 로컬 저장소에 평문으로 보관되므로 공용 PC나 공유 계정에서는 저장하지 않는 것을 권장합니다. 서버 파일과 실행 로그에는 키를 남기지 않습니다.

진행 중인 화면 요청을 취소할 수 있지만, 이미 GMS에 전달된 유료 작업은 결과 유실을 막기 위해 서버에서 끝까지 처리하고 로컬에 저장합니다. 코드 수정에 따른 HMR이나 브라우저 새로고침으로 연결이 끊겨도 완료된 결과는 기록에 남습니다.

Gemini 이미지 모델이 `NO_IMAGE`로 종료되면 시스템 프롬프트에 “이미지 생성 프롬프트를 작성해 텍스트로 출력”하는 지시가 있는지 확인하세요. 이런 프롬프트 작성 작업은 텍스트 탭에서 실행하고, 완성된 이미지 묘사만 이미지 탭에 입력해야 합니다. 화면에서도 충돌 가능성을 감지해 호출 전에 경고합니다.

## 주요 기능

- GPT·Gemini 이미지 생성, OpenAI·Gemini·Claude 텍스트 생성, GPT-4o mini TTS 음성 합성
- 키 확인 시 GMS가 제공하는 전체 텍스트 모델을 동적으로 조회하고 공급자별 검색·선택
- 목록 API에서 빠진 텍스트 모델은 공급자와 GMS 모델 ID로 직접 추가
- 선택한 모델의 전체 완료 시간과 API 생성 시간을 P50·P95로 비교
- 텍스트는 TTFT와 토큰 생성 속도, 이미지는 분당 처리량을 별도 집계
- TTS는 첫 오디오 바이트 시간, 수신 완료 시간, 입력 문자 처리량과 파일 크기를 별도 집계
- 공급자 응답의 실제 Input·Output·총 사용 토큰과 실행 전후 GMS 실제 잔액 차이 표시·저장
- 이미지·텍스트 모델에 로컬 이미지를 최대 4개까지 컨텍스트로 전달
- 이미지 비율을 먼저 선택하고 용도에 맞는 자주 쓰는 출력 크기 선택
- 모델을 선택하면 공급자 기본·권장 파라미터를 자동 적용하고, 필요할 때만 직접 덮어쓰기
- **현재 API 전달값**에서 각 `자동`·`auto` 옵션이 생략되는지 또는 문자열 그대로 전달되는지 확인
- 동시 요청 1개에서는 실행별 실제 차감량을 기록하고, 병렬 실행에서는 배치 전체 차감량 표시
- 반복·동시 실행, 진행 중 요청 취소
- 공급자 응답 요약과 전체 실행 JSON 확인·다운로드, 생성 이미지·음성 다운로드
- 기록 서버 페이지네이션(페이지당 10·25·50·100건), 검색·필터·재사용·삭제, 전체 검색 결과 CSV/JSON 내보내기

Claude Opus 4.1을 포함한 Claude 모델은 멀티모달 입력을 처리할 수 있지만 이미지 파일을 생성하는 모델은 아니므로 텍스트 탭에서 사용합니다. `whisper-1`은 음성 인식(STT) 모델이므로 현재 TTS 탭에는 포함하지 않습니다.

모델 안내에 적힌 단일 크레딧 값은 청구 단위가 명시되지 않아 실제 비용으로 간주하지 않습니다. 로컬 기록에는 공급자 응답의 전체 토큰을 **GMS 사용 토큰**으로 표시합니다. 동시 요청을 1개로 두면 각 실행 직전과 직후 GMS 잔액 차이를 해당 기록의 `actualCredit`에 저장합니다. 잔액 API가 실패하면 `측정 실패`와 오류 원인을 기록하며 0 Credit으로 처리하지 않습니다. 병렬 실행은 어느 요청이 얼마를 사용했는지 구분할 수 없으므로 실행별로 임의 분배하지 않고 배치 전체 차감량만 화면에 표시합니다.

숫자 파라미터의 **자동**은 해당 필드를 API 요청에서 생략해 선택 모델의 기본값을 사용한다는 뜻입니다. 이미지 옵션의 `auto`는 `size: "auto"`처럼 API에 실제 값으로 전달될 수도 있습니다. 화면의 **현재 API 전달값** 카드에서 두 경우를 구분해 볼 수 있습니다. Anthropic의 `max_tokens`처럼 필수이지만 공급자 기본값이 없는 항목은 GMS Quick Start 예제값을 사용하며, **공급자 기본값으로 복원** 버튼으로 언제든 초기화할 수 있습니다.

## 로컬 기록

- 실행 기록과 텍스트 결과: `data/runs/`
- 생성 이미지: `data/images/`
- 생성 음성: `data/audio/`
- 업로드한 컨텍스트 이미지: `data/context/`

실제 결과 파일은 Git에서 제외됩니다. 화면에서 기록을 삭제하면 연결된 생성 결과와 컨텍스트 이미지도 함께 삭제됩니다. 공급자 응답의 이미지 Base64 원문은 JSON 크기를 줄이기 위해 생략하고 이미지 파일로 저장합니다.

기록 목록은 기본 25건씩 표시되며 페이지당 10·25·50·100건으로 바꿀 수 있습니다. 검색과 상태 필터는 현재 페이지가 아닌 해당 테스트 종류의 전체 기록에 적용되고, CSV·JSON 내보내기도 검색 결과 전체를 포함합니다.

## 점검

```powershell
pnpm check
```

이 명령은 타입 검사, 린트, 테스트, 프로덕션 빌드를 차례로 실행합니다. 실제 GMS 생성 요청은 크레딧이 발생하므로 자동 점검에는 포함하지 않습니다.
